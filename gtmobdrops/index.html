GTMobDrops 快速开始 ​

GTMobDrops 是完整怪物掉落系统。它可以把掉落直接写在 MythicMobs 怪物配置里，也可以把大量掉落拆到 drops/、droppools/、sets/、rateBoost/ 文件夹中分类管理。

这一页先带你做出第一个能掉落的怪物。后面的页面再讲掉落池、套装暗改、属性加成和复杂动作。

先记住概率写法 ​

GTMobDrops 的掉落概率使用 0.0 ~ 1.0 小数。

text
1.0  = 100%
0.5  = 50%
0.1  = 10%
0.05 = 5%
0.01 = 1%

所以：

yaml
- "DIAMOND 1 1.0"  # 100% 掉 1 个钻石
- "DIAMOND 1 0.1"  # 10% 掉 1 个钻石

首次测试建议先使用 1.0，确认链路正常后再改成正式概率。

安装步骤 ​

  1. 把 GTCore.jar 放进服务器 plugins 文件夹。
  2. 把 GTMobDrops.jar 放进服务器 plugins 文件夹。
  3. 如果你要在 MythicMobs 怪物里写 GTDrops，服务器也要安装 MythicMobs。
  4. 启动服务器一次，让插件生成配置。
  5. 在 plugins/GTCore/config.yml 里填写 QQ 和 GTMobDrops 激活码。
  6. 重启服务器。
  7. 控制台看到 GTMobDrops 正常启用后，再开始写掉落。

常用命令：

text
/gtmd reload
/gtmd preview <怪物ID> [次数]
/gtmd debug

/gtmobdrops、/gtmd、/gmd 是同一个插件命令。

生成后的目录 ​

首次启动后主要看这些：

text
plugins/GTMobDrops/
  config.yml
  drops/
  droppools/
  rateBoost/
  sets/
位置用途
config.yml主配置，开关、消息、存储、保护、属性加成等
drops/独立怪物掉落，可以无限创建子文件夹
droppools/掉落池，可以无限创建子文件夹
rateBoost/独立物品加成，可以无限创建子文件夹
sets/套装暗改，可以无限创建子文件夹

旧版的 droppools.yml、sets.yml、rateBoost.yml 仍兼容。新服建议直接用文件夹分类。

方法一：写在 MythicMobs 怪物里 ​

适合 MythicMobs 怪物。怪物和掉落在同一个文件里，维护更直观。

打开或创建：

text
plugins/MythicMobs/Mobs/TestMob.yml

写入：

yaml
TestMob:
  Type: ZOMBIE
  Display: '&a测试怪物'
  Health: 20
  Damage: 2
  GTDrops:
    drops:
      - "DIAMOND 1 1.0"

效果：

  • 召唤 TestMob。
  • 玩家击杀它。
  • 100% 掉落 1 个钻石。

确认能掉后，把 1.0 改成正式概率：

yaml
GTDrops:
  drops:
    - "DIAMOND 1 0.1"

效果变成：10% 概率掉 1 个钻石。

数量可以写范围：

yaml
GTDrops:
  drops:
    - "DIAMOND 1-3 0.1"

效果变成：10% 概率掉 1 到 3 个钻石。

方法二：写在 GTMobDrops 的 drops 文件夹 ​

适合原版怪物，或者你不想把掉落写进 MythicMobs 文件。

创建文件：

text
plugins/GTMobDrops/drops/novice/zombie.yml

写入：

yaml
ZOMBIE:
  enabled: true
  drops:
    - "DIAMOND 1 1.0"

效果：

  • 玩家击杀原版僵尸。
  • 100% 掉落 1 个钻石。

确认生效后，把概率改成 10%：

yaml
ZOMBIE:
  enabled: true
  drops:
    - "DIAMOND 1 0.1"

物品 ID 怎么写 ​

原版物品可以直接写材料名：

yaml
drops:
  - "DIAMOND 1 1.0"
  - "GOLD_INGOT 3 0.5"

物品库物品带前缀：

yaml
drops:
  - "ni:gold_coin 1-5 1.0"
  - "mm:boss_soul 1 0.05"
  - "ia:namespace:item_id 1 0.1"
  - "oa:item_id 1 0.1"
  - "sx:item_id 1 0.1"

常用前缀：

前缀说明
不写前缀原版物品，例如 DIAMOND
ni:NeigeItems
mm:MythicMobs 物品
ia:ItemsAdder
oa:Oraxen
sx:SX-Item / SXItem / VirtuaItem

重载和测试 ​

改完配置后执行：

text
/gtmd reload

然后模拟掉落：

text
/gtmd preview ZOMBIE 200

如果是 MM 怪物：

text
/gtmd preview TestMob 200

这里的 200 是模拟 200 次掉落，不会真的刷 200 只怪。

测试顺序建议：

  1. 先用 1.0 概率。
  2. /gtmd reload。
  3. /gtmd preview 怪物ID 20。
  4. 实际召唤怪物击杀一次。
  5. 成功后再把概率改成正式值。

常见配置问题 ​

问题正确做法
把 10% 写成 10GTMobDrops 里 10% 写 0.1
把 100% 写成 100100% 写 1.0
YAML 缩进乱了每级两个空格，不用 Tab
MM 怪物没触发GTDrops 必须在怪物节点下面
物品库物品不掉先测试原版 DIAMOND，再测试物品库 ID
改完没效果执行 /gtmd reload，MM 文件也要重载
不是玩家击杀配置要求玩家击杀时，环境伤害不会触发
