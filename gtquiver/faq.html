常见问题 ​

GT 系列插件交流群：985955382

插件没有启用 ​

  • 确认 GTCore 已经正常启用。
  • 确认 plugins 目录中只有一个 GTQuiver Jar。
  • 更换 Jar 后完整重启，不使用热加载。
  • 从控制台最早出现的 GTQuiver 报错开始看。

/gtq reload 失败 ​

常见原因：

  • YAML 缩进或引号错误。
  • 箭袋 ID、箭 ID 大小写重复。
  • 当前版本不存在配置中的材质。
  • game-modes 写错。
  • lore-regex 正则错误。
  • attribute-bridge.provider 不是 CA、AP、BOTH、NONE。
  • provider-item 对应物品不存在。

修正后执行 /gtq reload，再用 /gtq list 查看已加载 ID。

主手右键空气打不开 ​

正确操作是主手拿箭袋，按住 Shift 右键空气或方块。

副手不会打开。仍无反应时检查箭袋权限、世界、游戏模式和 GTCore 条件。

地上的箭袋无法拾取 ​

先看箭袋 Lore 是否显示了其他玩家的绑定信息。绑定箭袋只能由绑定玩家拾取和使用，管理员权限不会绕过绑定。

需要公共箭袋时重新发放非绑定版本：

text
/gtq give 玩家 箭袋ID 1 unbound

玩家改名不影响绑定，归属仍按 UUID 判断。

背包没有箭时无法拉弓 ​

  • 箭袋需要放在 settings.equipped-slots 指定位置。
  • 普通弓只能使用普通箭、药水箭、光灵箭。
  • 烟花只供弩使用。
  • 玩家需要满足箭袋权限和使用条件。
  • 背包内存在其他兼容箭时，会按原版规则使用背包箭。

执行 /gtq debug 玩家 可查看当前箭袋和第一支可用箭。

轻点右键后背包出现箭 ​

确认使用当前版本 GTQuiver，并完整重启服务器。

仍能复现时开启 settings.debug，保留一次操作产生的日志，同时检查反作弊、背包同步、物品锁定类插件。

箭袋不扣箭 ​

  • 创造模式默认不消耗。
  • 普通原版箭会遵守无限附魔。
  • 背包已有箭时，可能消耗的是背包箭。

关闭无限附魔处理：

yaml
settings:
  respect-infinity: false

彩色公式不生效 ​

实际伤害需要写在 effects.formula-damage：

yaml
effects:
  formula-damage:
    - "&7物理伤害: &c+20000"

下面只是一行显示 Lore，不会自动增加 GTQuiver 伤害：

yaml
item:
  lore:
    - "&7造成伤害20000点"

带计算公式的写法：

yaml
formula-damage:
  - "&7物理伤害: &c+{calc:120+%player_level%*2}"

公式仍无结果时开启 debug，检查 PAPI 变量是否返回数字。

AP 箭拿在手里有属性，射出去没有 ​

yaml
attribute-bridge:
  provider: "AP"

默认值是 CA，AP 服务器需要手动修改。修改后完整重启，再执行 /gtq debug 玩家 查看 AP 是否已连接。

CA 箭没有属性 ​

yaml
attribute-bridge:
  provider: "CA"

确认箭本身符合当前 CA 的物品属性格式，并用 /gtq debug 玩家 查看 CA 状态。

CA 和 AP 同时安装后伤害翻倍 ​

检查是否使用：

yaml
provider: "BOTH"

只使用一套属性时改为 CA 或 AP。

自动换箭后属性没有变化 ​

确认：

  • attribute-bridge.provider 选择正确。
  • 两种箭的实际 Lore、属性或 NBT 不同。
  • 使用当前版本 GTQuiver。
  • 其他武器或属性插件没有另外缓存属性。

问题日志中同时附上 /gtq debug 玩家 输出。

弓左键伤害异常 ​

默认禁止弓和弩近战：

yaml
combat:
  block-bow-melee: true

需要允许弓弩左键伤害时改为 false。GTQuiver 自定义箭始终不能手持近战。

射出的箭无法拾取 ​

默认配置就是禁止回收：

yaml
arrow-recovery:
  allow-non-infinity: false

允许回收：

yaml
arrow-recovery:
  allow-non-infinity: true

无限弓射出的箭、创造模式未消耗的箭仍不可拾取。

拾取后没有进入箭袋 ​

  • allow-non-infinity 需要为 true。
  • 箭袋需要装备在配置槽位。
  • 箭要符合箭袋 accept。
  • 箭袋需要有空位或可合并堆叠。
  • 玩家自动收箭需要开启。

有权限时执行：

text
/gtq autopickup on

箭袋无法接收时，箭进入普通背包。

/gtq autopickup 没有权限 ​

权限默认不授予：

text
gtquiver.command.autopickup

LuckPerms：

text
/lp user 玩家 permission set gtquiver.command.autopickup true

PAPI 变量返回未选择 ​

text
%gtquiver_current_arrow%
%gtquiver_current_arrow_colored%

没有装备箭袋、箭袋为空、箭袋不可用时都会返回 未选择。

同时确认 PlaceholderAPI 已启用，并在安装或更新后完整重启服务器。

Shift+左键不能切换箭 ​

需要同时满足：

  • 玩家处于潜行状态。
  • 左键空气或方块。
  • 已装备可用箭袋。
  • 箭袋内至少有两种不同箭。

完整物品相同、只有数量不同的箭算同一种。

跨服后箭袋无法读取 ​

  • 跨服背包插件需要保留完整物品 NBT。
  • 两端 GTCore、GTQuiver 版本保持一致。
  • 两端箭袋 ID 和容量保持一致。
  • 目标服容量不能小于箭袋原容量。
  • 自定义箭使用的物品库和属性插件保持一致。
