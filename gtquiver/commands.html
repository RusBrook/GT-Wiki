命令与权限 ​

主命令：

text
/gtquiver

别名：

text
/gtq
/quiver
/jiandai

命令表 ​

命令权限说明
/gtq无查看帮助
/gtq opengtquiver.command.open打开手持或装备的箭袋
/gtq list无查看箭袋和自定义箭 ID
/gtq autopickup [on|off]gtquiver.command.autopickup切换自动收箭
/gtq give <玩家> <箭袋ID> [数量] [bound|unbound]gtquiver.command.give发放绑定或非绑定箭袋
/gtq givearrow <玩家> <箭ID> [数量]gtquiver.command.give发放自定义箭
/gtq reloadgtquiver.admin重载配置
/gtq debug [玩家]gtquiver.admin查看箭袋、射击和属性状态

/gtq open ​

text
/gtq open

优先打开当前手持箭袋，手中没有箭袋时按 settings.equipped-slots 查找。

只能由玩家执行。

/gtq list ​

text
/gtq list

显示 quivers.yml 和 arrows.yml 当前已加载的 ID。

/gtq autopickup ​

text
/gtq autopickup
/gtq autopickup on
/gtq autopickup off

不带参数时切换状态。玩家默认开启自动收箭。

该命令只控制拾取后是否自动进入装备箭袋。全局是否允许回收仍由以下配置决定：

yaml
arrow-recovery:
  allow-non-infinity: false

权限默认不授予：

text
gtquiver.command.autopickup

LuckPerms 示例：

text
/lp user 玩家 permission set gtquiver.command.autopickup true

/gtq give ​

text
/gtq give <玩家> <箭袋ID> [数量] [bound|unbound]

数量默认 1，范围 1 - 64。绑定模式：

模式效果
bound绑定给接收玩家
unbound不绑定，默认

示例：

text
/gtq give Player survival 1 bound
/gtq give Player ranger 2 unbound

背包放不下的部分掉落在目标玩家脚下。绑定箭袋只有接收玩家可以拾取和使用。

每个箭袋内容独立，不会合并成普通堆叠物品。

/gtq givearrow ​

text
/gtq givearrow <玩家> <箭ID> [数量]

别名：

text
/gtq arrow <玩家> <箭ID> [数量]

数量默认 1，范围 1 - 2304。插件按物品最大堆叠数发放，背包放不下的部分掉落在脚下。

/gtq reload ​

text
/gtq reload

重载：

text
config.yml
quivers.yml
arrows.yml
messages.yml

配置有误时继续使用上一份有效配置，错误原因查看控制台。

安装、卸载或更新 GTCore、CA、AP、MythicMobs、PlaceholderAPI、物品库后，使用完整重启。

/gtq debug ​

text
/gtq debug
/gtq debug <在线玩家>

显示：

  • 当前配置的 CA/AP 类型。
  • CA、AP 是否已连接。
  • 箭回收和自动收箭状态。
  • 玩家装备箭袋的槽位和 ID。
  • 箭袋总数和当前可用箭。
  • 当前射击、属性、弩装填状态数量。

需要控制台详细日志时再开启：

yaml
settings:
  debug: true

权限 ​

权限默认说明
gtquiver.use全部玩家默认箭袋使用权限
gtquiver.command.open全部玩家使用 /gtq open
gtquiver.command.autopickup无切换自动收箭
gtquiver.command.giveOP发放箭袋和自定义箭
gtquiver.adminOP重载和调试
gtquiver.bypass.conditionOP绕过箭袋使用条件，不绕过箭袋独立权限

箭袋还可以在 quivers.yml 中设置独立 permission。
