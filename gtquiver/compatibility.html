物品库与属性插件

本页目录

属性、技能、物品库与变量 ​

CraneAttribute ​

yaml
attribute-bridge:
  provider: "CA"

默认使用 CA。箭上的属性按服务器当前 CA 物品格式填写，GTQuiver会使用箭袋中实际射出的箭参与伤害计算。

箭本身无法被 CA 识别时，放入箭袋后也不会产生属性。

AttributePlus Pro ​

yaml
attribute-bridge:
  provider: "AP"

AP 服务器必须把默认的 CA 改成 AP。

常见情况：箭拿在手里能触发 AP，射出去没有属性。先检查 provider，再执行：

text
/gtq debug 玩家

确认 AP 已连接。

同时安装 CA 和 AP ​

yaml
attribute-bridge:
  provider: "BOTH"

BOTH 会让两套属性插件都处理箭。相同属性被两边同时识别时，伤害会叠加。

只用一套属性时明确写 CA 或 AP。不使用属性插件时写：

yaml
attribute-bridge:
  provider: "NONE"

属性 Lore 与公式伤害 ​

yaml
item:
  lore:
    - "&7物理伤害: &c+20000"
effects:
  formula-damage:
    - "&7物理伤害: &c+20000"

上面两处不是同一项：

  • item.lore 由 CA/AP 按自己的格式决定是否生效。
  • effects.formula-damage 由 GTQuiver 计算。

两边都生效时会叠加。

MythicMobs ​

yaml
effects:
  mythic-skills:
    - "GTQ_FlameImpact"

命中实体或方块时施放技能。MythicMobs 为可选插件，没有安装时不影响 GTQuiver 基础功能。

技能伤害会与 CA/AP、damage-multiplier、additional-damage、formula-damage 叠加。

GTCore 物品提供器 ​

箭袋和自定义箭都支持：

yaml
provider-item: "提供器前缀:物品ID"

例如：

yaml
provider-item: "mm:flame_arrow"

可用前缀以 GTCore 当前接入的物品库为准。物品 ID 无效时，发放命令会提示 item-create-failed。

物品库提供的名称、Lore、附魔、模型和 NBT 会完整保留。

PlaceholderAPI ​

需要安装 PlaceholderAPI。

当前箭名称 ​

text
%gtquiver_current_arrow%

返回去色后的当前箭名称。

当前箭彩色名称 ​

text
%gtquiver_current_arrow_colored%

返回物品原始彩色名称。

以下情况返回 未选择：

  • 没有装备箭袋。
  • 箭袋为空。
  • 当前箭袋没有使用权限或不满足条件。
  • 箭袋数据无法读取。

变量按 settings.equipped-slots 的顺序读取第一个箭袋。

跨服 ​

跨服背包插件需要保留完整物品 NBT。互通子服保持以下内容一致：

text
GTCore、GTQuiver 版本
quivers.yml 中的箭袋 ID 和容量
arrows.yml 中的自定义箭配置
CA/AP 选择和属性配置
物品库配置

player-settings.yml 保存自动收箭开关。未同步该文件时，每个子服分别记录。
