UI、HUD 与按键 ​

GTSkillView 的 UI 分为两层：

  1. 服务端逻辑：技能、槽位、MP、冷却、按键、动作。
  2. 客户端展示：PaiUI、ArcartX、KongCore、DragonCore 或原版 GUI。

客户端 UI 只负责显示和回传动作。释放、绑定、升级、扣蓝、冷却、权限、条件都由服务端统一处理。

文件位置 ​

text
plugins/GTSkillView/client-ui/
  providers.yml
  ax/
    main.yml
    hud.yml
  dragoncore/
    main.yml
  kongcore/
    main.yml
  paiui/
    KeyConfig.yml
    webgui/gtskillview/
      main.html
      hud.html
      style.css
      ui.js

client-ui 下的模板用于服务器侧注册和同步客户端界面。
更新 UI 模板后，需要手动覆盖旧文件，或删除旧文件后重启生成。

providers.yml 路由顺序 ​

yaml
routing:
  fallback-order:
    - paiui
    - ax
    - kongcore
    - dragoncore
    - vanilla

fallback-order 控制 ui.default: auto 时的选择顺序。

值说明
paiuiPaiUI / PaiDaXing WebGui
axArcartX
kongcoreKongCore
dragoncoreDragonCore
vanilla原版箱子 GUI

只使用 AX：

yaml
routing:
  fallback-order:
    - ax
    - vanilla

只使用原版 GUI：

yaml
routing:
  fallback-order:
    - vanilla

providers 开关 ​

yaml
providers:
  paiui:
    enabled: true
  ax:
    enabled: true
  kongcore:
    enabled: true
  dragoncore:
    enabled: true

关闭某个 UI：

yaml
providers:
  dragoncore:
    enabled: false

关闭后 auto 路由会跳过该 UI。

统一动作协议 ​

客户端 UI 回传动作时使用逻辑动作，不直接执行技能结果。

常见动作：

text
key SLOT_1
cast 1
bind 1 thunder_slash
unbind 1
upgrade thunder_slash
reset
open

按键释放推荐走：

text
key SLOT_1

服务端会读取：

text
config.yml -> release.client.bindings.SLOT_1

再决定它对应哪个槽位或动作。

默认按键 ​

yaml
release:
  client:
    bindings:
      SLOT_1:
        slot: 1
        keys:
          paiui: ["R"]
          ax: ["R"]
          kongcore: ["R"]
          dragoncore: ["R"]
      SLOT_2:
        slot: 2
        keys:
          paiui: ["T"]
          ax: ["T"]
          kongcore: ["T"]
          dragoncore: ["T"]
      SLOT_3:
        slot: 3
        keys:
          paiui: ["Z"]
          ax: ["Z"]
          kongcore: ["Z"]
          dragoncore: ["Z"]
      SLOT_4:
        slot: 4
        keys:
          paiui: ["X"]
          ax: ["X"]
          kongcore: ["X"]
          dragoncore: ["X"]
      SLOT_5:
        slot: 5
        keys:
          paiui: ["C"]
          ax: ["C"]
          kongcore: ["C"]
          dragoncore: ["C"]

默认键位：

槽位键
1R
2T
3Z
4X
5C

改键只改 release.client.bindings。不要在模板里把物理键写死。

providers:
  paiui:

    channel: "gtskillview:update"
    action-channel: "gtskillview:action"
    data-route: "gtskillview.data"
    action-route: "gtskillview.action"
    mirror-webgui: true
    mirror-keyconfig: true
    mirror-keyconfig-overwrite: false
    mirror-keyconfig-merge-missing: true
    dynamic-items: true
    dynamic-item-namespace: ""
    dynamic-items-on-hud: false

      hud: "layout/gtskillview/hud.html"
字段作用

PaiUI 文件同步 ​

WebGui 默认同步到：

text
plugins/PaiDaXingPlugins/advanced/WebGui/layout/gtskillview/

按键默认同步到：

text
plugins/PaiDaXingPlugins/advanced/KeyConfig/KeyConfig.yml

mirror-keyconfig-overwrite: false 时，只补缺失项，不覆盖已有同名按键。
如果旧按键残留，先备份 KeyConfig，再删除 GTSkillView 生成过的按键项。

PaiUI 页面约定 ​

页面入口使用：

text
window.PaiDaXing
window.PDX

服务端通信使用：

text
PaiDaXing.server.request(...)
PaiDaXing.server.notify(...)

HUD 非交互区域保持不可点击，避免挡住玩家视角和鼠标操作。
动态物品和真实槽位不要每帧刷新。列表变化、打开界面、数据变化时刷新即可。

PaiUI 图标 ​

技能图片优先读取：

yaml
texture: "textures/gtskillview/thunder_slash.png"

展示物品可使用动态物品。普通技能 HUD 更推荐使用 texture，减少动态物品刷新成本。

AX 官方 API 核对 ​

AX v2 服务端 API 统一入口：

java
ArcartXAPI.getUIRegistry()
ArcartXAPI.getKeyBindRegistry()

GTSkillView 的主界面和 HUD 走 ArcartXUIRegistry：

API用途
getUIRegistry().register(id, src)注册主界面和 HUD
getUIRegistry().open(player, id)打开已注册 UI
getUIRegistry().sendPacket(player, id, handler, packet)向 UI 的 packetHandler 推送数据

register(id, src) 的 src 支持 File、YAML 字符串、YamlConfiguration。传字符串时内容必须是完整 YAML，不是文件路径。

UI 数据推送对应模板中的 ui.packetHandler：

yaml
ui:
  packetHandler:
    update: |-
      var.mana = packet['mana']

服务端推送 handler 名称为 update 时，模板中必须存在 packetHandler.update。

AX 按键文件与 API ​

GTSkillView 会按 release.client.bindings 生成：

text
plugins/ArcartX/key_bind/simple_key/GTSkillView.yml

示例：

yaml
GTSkillView_SLOT_1:
  keys:
    - "R"
  trigger:
    - "让玩家发送消息{message = /gtsv key SLOT_1 ax}"

默认按键监听走 AX 的 key_bind/simple_key。这类按键由服务端统一配置，玩家不能在客户端按键设置里自行改键。

AX API 中按键分两类：

类型说明适用场景
SimpleKey服务端定义的固定按键组合统一配置快捷键
ClientKey出现在客户端按键设置里的按键玩家可自行调整键位

默认键位完全来自 release.client.bindings.*.keys.ax。R、T、Z、X、C 只是默认值，按 AX 按键 ID 改配置即可替换。

AX 也提供 key_bind/client_key 配置和 registerClientKeyBind(keyName, category, defaultKey, ...) API。启用 ClientKey 模式时，按键会出现在客户端按键设置中；触发动作需要通过 key_bind/key_group 或 API 回调接回 GTSkillView。

主界面和 HUD 的 clickLeft、releaseLeft 是 UI 控件触发器，不是物理键盘按键监听。技能释放按键不要绑到控件左键事件上，避免普通攻击、界面点击、拖拽动作误触发释放。

AX 客户端图片 ​

AX 客户端图片放在 AX 客户端资源目录。

示例路径：

text
.minecraft/resourcepacks/ArcartX/resource/gtskillview/skill/panel_base_940x560.png
.minecraft/resourcepacks/ArcartX/resource/gtskillview/hud/01_bar_shell_72x412.png
.minecraft/resourcepacks/ArcartX/resource/textures/gtskillview/thunder_slash.png

技能图标的 texture 路径按客户端资源包实际结构放置。
服务器只负责下发 slot_1_texture、skill_1_texture 等变量。

AX PACKET 安全 ​

AX 客户端发包只接受明确标识：

text
gtskillview
gtskillview:action

Packet.send("gtskillview", "bind 1 thunder_slash", "main") 这类包可以处理。
空标识符包只有第一个参数明确是 gtskillview 时才处理。普通鼠标事件、槽位文本、数字不会被当成技能释放。

这样可以避免左键攻击、普通 UI 包、其他插件包误触发 GTSkillView。

AX 的客户端通信事件会提供数据包 ID 和参数列表。GTSkillView 只处理动作包，不把普通 UI 点击、普通槽位事件或无标识数据当技能动作。

控件触发器使用 action 节点：

yaml
action:
  clickLeft: |-
    Packet.send("gtskillview", "bind 1 " + var.selectedSkillId, "main")

clickLeft、releaseLeft 属于 AX 控件触发器。拖拽绑定可以使用释放事件尝试处理，但不同 UI 结构下仍应保留“点击技能，再点击槽位”的稳定绑定链路。

AX 主界面绑定 ​

AX 主界面支持：

  1. 点击技能卡选中技能。
  2. 点击左侧槽位绑定当前技能。
  3. 点击右侧绑定按钮绑定到 1 号槽。
  4. 按住技能卡拖到槽位松开，尝试走释放事件绑定。

不同 AX 版本对拖拽/松开事件支持不完全一致。点击选中再点击槽位是稳定链路。

资源更新注意 ​

GTSkillView 使用默认资源保存机制：

text
文件不存在 -> 生成默认资源
文件已存在 -> 不覆盖

更新插件后，如果客户端 UI 模板没有变化，检查：

text
plugins/GTSkillView/client-ui/ax/main.yml
plugins/GTSkillView/client-ui/ax/hud.yml
plugins/GTSkillView/client-ui/paiui/webgui/gtskillview/

处理方式：

  1. 备份旧文件。
  2. 删除旧模板。
  3. 重启服务器让插件重新生成。
  4. 或直接用新模板覆盖旧模板。

排查流程 ​

按键不生效：

  1. 检查 config.yml -> release.client.enabled。
  2. 检查 release.client.allow-providers 是否包含当前来源。
  3. 检查 release.client.bindings 是否有对应绑定 ID。
  4. PaiUI 检查 advanced/KeyConfig/KeyConfig.yml。
  5. AX 检查 key_bind/simple_key/GTSkillView.yml。
  6. 用 /gtsv key SLOT_1 ax 直接测试服务端解析。

HUD 挡鼠标：

  1. 确认打开的是 HUD 文件，不是主界面文件。
  2. AX 检查 ui.isHud: true。
  3. PaiUI HUD 非交互区域使用不可点击样式。
  4. 主界面和 HUD 不要共用同一个交互模板。

图标不显示：

  1. 原版 GUI 检查 icon。
  2. 客户端 UI 检查 texture。
  3. 检查客户端资源包是否有对应图片。
  4. 检查路径大小写。
  5. AX 图片检查客户端资源目录。
