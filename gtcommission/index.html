GTCommission 快速开始 ​

GTCommission 是每日委托、周期任务和玩家悬赏插件。生存服可以用它做挖矿、钓鱼、收集物资、周常和玩家求购；MMORPG 服务器可以接 MythicMobs、GTMobDrops、GTGather 和副本通关任务。

委托与悬赏是两套独立入口：

入口用途
/gtdaily打开默认委托面板
/gtdaily <面板ID>打开指定委托面板
/gtbounty打开悬赏大厅
/gtbounty publish发布服主开放的悬赏
/gtcom打开默认委托面板

安装要求 ​

  • Minecraft 1.12.2 - 1.21.11。
  • GTCore 1.0.49 或更高版本。
  • 使用 PAPI 条件、自定义经济或 PAPI 显示时安装 PlaceholderAPI。
  • 使用金币悬赏时安装 Vault 和任意 Vault 经济插件。
  • 使用 PlayerPoints、DreamPoints 时安装对应插件。

安装步骤 ​

  1. 把 GTCore 和 GTCommission 放进服务端 plugins 文件夹
  2. 启动服务器一次
  3. 确认控制台显示 GTCommission 已启用
  4. 输入 /gtdaily 检查默认委托面板
  5. 输入 /gtbounty 检查悬赏大厅
  6. 修改配置后输入 /gtcom reload

首次启动后生成：

text
plugins/GTCommission/
  config.yml
  messages.yml
  tasks/
  pools/
  boards/
  bounties/
  guis/
    commission/
    bounty/
  data/

这些目录都支持继续创建子目录。任务多时可以按玩法分类：

text
tasks/
  survival/
    mining.yml
    farming.yml
  rpg/
    mythic.yml
    dungeon.yml
  events/
    summer.yml

文件名不决定任务 ID。任务 ID 以配置节点名为准，同一个目录中不能出现重复 ID。

默认内容 ​

默认配置包含：

  • 生存挖矿、狩猎、钓鱼、合成、提交原木。
  • MythicMobs 怪物、GTMobDrops 物品、GTGather 采集、副本通关。
  • PAPI 条件提交任务。
  • 每日、每周、每月和 NPC 支线面板示例。
  • 怪物讨伐、玩家追缉、物资征集悬赏示例。

默认只开启 default 委托面板、怪物讨伐悬赏和物资征集悬赏。周常、月常、NPC 示例和玩家追缉默认关闭。

第一个独立 NPC 委托 ​

创建 boards/blacksmith.yml：

yaml
enabled: true
name: "&6铁匠委托"
gui: default
tasks: [submit_logs]
selection-count: 1
conditions: []
refresh:
  mode: DAILY
  timezone: Asia/Shanghai
  hour: 4
  minute: 0

让 NPC 执行：

text
gtdaily open blacksmith %player%

这个面板拥有自己的刷新时间、任务进度、连续完成次数和 GUI。它不会和默认 /gtdaily 面板混在一起。
