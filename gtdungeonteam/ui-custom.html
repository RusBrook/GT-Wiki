GUI 与客户端界面 ​

GTDungeonTeam 有两层界面：

  • gui/：原版背包 GUI，所有服务器都能用。
  • client-ui/：PaiUI、AX、DragonCore、KongCore、Dialog 等客户端界面。

客户端 UI 模式由服主明确指定。指定引擎未安装、未启用或 API 初始化失败时，插件直接打开原版 GUI。

选择默认界面 ​

yaml
ui:
  default: vanilla
  prefer-client-ui: true

可选：

text
vanilla
paiui
ax
dragoncore
kongcore
dialog

prefer-client-ui: true 表示启用 ui.default 指定的客户端 UI。失败后走原版 GUI。

客户端 UI 配置 ​

文件：

text
plugins/GTDungeonTeam/client-ui/providers.yml

插件不会按顺序探测 UI 引擎，也不会在玩家打开菜单时扫描插件或 JAR。启动或 /gdt reload 时只检查 ui.default 指定的前置：

ui.default检查的前置
vanilla无
paiuiPaiUI 或 PaiDaXingPlugins
axArcartX
dragoncoreDragonCore
kongcoreKongCore、KongCorePlugin 或配置的插件名
dialog无，要求服务端版本支持原版 Dialog

providers.yml 只保存各引擎自身的连接参数、页面 ID 和资源文件路径，不负责选择运行模式。

每个引擎都有自己的 views：

yaml
providers:
  paiui:
    views:
      main: "layout/gtdungeonteam/main.html"
      team: "layout/gtdungeonteam/main.html"
      auction: "layout/gtdungeonteam/main.html"

  ax:
    views:
      main: "gtdungeonteam_main"
      team: "gtdungeonteam_team"
      auction: "gtdungeonteam_auction"

PaiUI 默认使用单页 main.html，在页面内部切换视图，减少多 HTML 切换闪屏。

每个文件一个页面。写法接近 TrMenu：

yaml
main:
  Title: "&8副本大厅 &7| &f<player>"
  Layout:
    - "#########"
    - "#   S   #"
    - "#       #"
    - "# T D I #"
    - "# A M R #"
    - "#########"
  Icons:
    S:
      Display:
        Material: DIAMOND_SWORD
        Name: "&c&l开始挑战"
        Lore:
          - "&7队伍: &f<team_name>"
          - "&7人数: &f<members>/<max_members>"
      Actions:
        - "start:"

Layout ​

Layout 一行 9 格。一页 6 行就是 54 格。

yaml
Layout:
  - "#########"
  - "#MMMMMMM#"
  - "#MMMMMMM#"
  - "#C#I#D#A#"
  - "#V#X#R#K#"
  - "L#######B"

每个字符对应 Icons 下的一个图标。
同一个字符出现多次，会放同一个图标。

Display ​

yaml
Display:
  Asset: team
  Material: CHEST
  Name: "&a队伍管理"
  Lore:
    - "&7队伍: &f<team_name>"
    - "&7人数: &f<members>/<max_members>"
  Glow: "<public>"
  ItemFlags:
    - HIDE_ENCHANTS
  CustomModelData: "{calc:10000+<members>}"
字段作用
Asset读取 assets/ 中的资源映射
Material原版材料
Name物品名
Lore物品说明
Glow是否发光，支持变量
ItemFlags物品 Flag
CustomModelData模型数据，支持公式

标题、物品名、Lore、Glow、CustomModelData 都会渲染变量。

Condition ​

yaml
Condition: "<is_leader>"

条件为 false 时不显示这个图标。

常用：

yaml
Condition: "<has_team>"
Condition: "!<has_team>"
Condition: "<is_leader>"

! 表示取反。

动态列表 Type ​

成员列表：

yaml
M:
  Type: member-list
  Display:
    Material: PAPER
    Name: "&f<member>"
    Lore:
      - "&7身份: &e<member_role>"
      - "&7在线: &f<member_online>"
      - "&7所在服: &f<member_server>"

常用动态类型：

Type用途
member-list队伍成员
dungeon-list副本列表
room-list公开房间
application-list入队申请
invite-list邀请候选玩家
requirement-template-list入队条件模板
damage-list结算伤害列表

动态列表的每个元素都有自己的上下文变量，例如 <member>、<dungeon_id>、<team>、<applicant>。

动作说明

  - "team:create"

资源图标 assets ​

GUI 里写：

yaml
Display:
  Asset: loot
  Material: HOPPER

资源文件：

text
plugins/GTDungeonTeam/assets/
  base.yml
  itemsadder.yml
  craftengine.yml
  oraxen.yml
  vanilla.yml

建议 GUI 里优先写稳定的 Asset，不同材质包在 assets/ 分开配置。这样以后从 ItemsAdder 换 Oraxen，不需要把所有 GUI 文件翻一遍。

PaiUI 文件 ​

text
plugins/GTDungeonTeam/client-ui/paiui/webgui/gtdungeonteam/
  main.html
  ui.css
  ui.js

默认配置会把 WebGui 文件同步到 PaiUI 可访问目录：

yaml
providers:
  paiui:
    mirror-webgui: true
    mirror-overwrite-newer-source: true

改完后执行：

text
/gdt reload

如果你手动改 PaiUI 目标目录里的文件，而源目录没改，下一次重载可能被同步覆盖。正式修改建议改 GTDungeonTeam 自己的 client-ui/paiui/webgui/ 源文件。

AX、DragonCore、KongCore ​

文件位置：

text
client-ui/ax/
client-ui/dragoncore/
client-ui/kongcore/

每个页面一个 yml：

text
main.yml
team.yml
invite.yml
applications.yml
requirements.yml
auction.yml
dungeons.yml
dungeon-detail.yml
rooms.yml
room-detail.yml
result.yml

这些文件的具体组件写法按对应 UI 引擎自己的规则来。GTDungeonTeam 负责打开、同步变量、接收动作。

DragonCore 默认页按 DragonCore 2.6.2.9 接口编写。每个页面必须有与 providers.yml 视图一致的 match，每个组件必须明确写 type：

yaml
match: "gtdungeonteam/main"

背景:
  type: "texture"
  texture: "gtdungeonteam/background.png"

标题:
  type: "label"
  texts: "副本作战大厅"

服务端固定调用 PacketSender.sendYaml、sendSyncPlaceholder、sendOpenGui，并通过 CustomPacketEvent 接收 方法.发包('GTDungeonTeam', ...)。页面文件在启动或重载时解析，玩家打开界面时不读文件、不扫描插件、不执行服务端指令。

重载注意 ​

text
/gdt reload

会重新读取 client-ui 和 gui。如果客户端仍看到旧页面：

  1. 确认改的是 plugins/GTDungeonTeam/client-ui/。
  2. 执行 /gdt reload。
  3. 关闭并重新打开界面。
  4. PaiUI 场景下检查是否同步到了 PaiUI 的 WebGui 目录。
  5. AX、DragonCore、KongCore 场景下检查对应客户端资源是否刷新。
  6. DragonCore 场景下确认客户端材质包包含页面里引用的 gtdungeonteam/ 贴图。
