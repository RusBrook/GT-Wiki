队伍、房间与入队条件 ​

GTDungeonTeam 的队伍系统可以单独使用。即使服务器没有装副本插件，也能做普通组队、公开房间、邀请、申请、掉落分配和经验共享。

队伍状态 ​

每个队伍有这些核心状态：

状态说明
队伍 ID内部唯一 ID，用于命令、跨服和存储
队伍名玩家看到的名称，队长可修改
队伍介绍公开房间和队伍详情展示文字，队长可修改
队长拥有邀请、踢人、设置、开始副本权限
成员队伍玩家列表
目标副本本次准备挑战的副本
公开状态是否出现在公开房间
申请状态是否接收入队申请
队友伤害副本内是否允许队友互伤
掉落分配队伍是否开启掉落分配开关
入队条件队长设置的申请门槛

创建队伍 ​

玩家输入：

text
/gdt team create

或打开 /gdt 主界面，点击队伍房间里的创建按钮。

新队伍会读取：

yaml
team:
  default-max-players: 4
  default-name: "<leader>的队伍"
  default-description: "欢迎加入队伍。"
  default-public: true
  default-allow-apply: true

<leader> 会替换成队长名称。

邀请玩家 ​

队长输入：

text
/gdt team invite 玩家名

玩家收到邀请后输入：

text
/gdt team accept

也可以打开邀请列表：

text
/gdt team invite

邀请会在 team.invite-expire-seconds 后过期。跨服场景下，邀请会写入共享存储，目标玩家在其他子服也能收到等待状态。

公开房间 ​

公开房间开关：

text
/gdt team public
/gdt team private

玩家查看房间：

text
/gdt rooms
/gdt rooms list

点击公开房间中的队伍，会先打开队伍详情。详情页显示：

  • 队伍名
  • 队伍 ID
  • 队长
  • 当前人数
  • 目标副本
  • 队伍介绍
  • 入队条件
  • 成员列表

看完再申请：

text
/gdt rooms join 队伍ID

如果队伍允许申请，玩家会进入申请列表；如果队伍允许直接加入并满足条件，会直接进队。

快速加入 ​

text
/gdt rooms quick

是否启用看配置：

yaml
team:
  public-room:
    enabled: true
    quick-join: true

快速加入适合轻量玩法。大型副本服建议让玩家先看队伍详情和条件，再申请。

申请列表 ​

队长打开申请列表：

text
/gdt team applications

同意：

text
/gdt team acceptapp 玩家名

拒绝：

text
/gdt team rejectapp 玩家名

申请会在 team.application-expire-seconds 后过期。单个队伍最多保留 team.max-applications 条申请。

队伍名和介绍 ​

队长改名：

text
/gdt team rename 星火开荒队

队长改介绍：

text
/gdt team desc 来一个奶，战力 5w 以上，打噩梦森林

GUI 和客户端 UI 里也有修改入口。原版 GUI 会临时读取玩家下一句聊天内容；PaiUI、AX、DragonCore、KongCore 这类客户端 UI 可以按各自输入能力做输入框或包回调。

输入取消：

text
cancel
取消

入队条件的设计 ​

配置中只定义“条件模板”，不要替队长写死每个队伍要多少战力。

yaml
team:
  join-requirements:
    templates:
      power:
        name: "战斗力"
        placeholder: "%player_zhandouli%"
        default-operator: ">="
      level:
        name: "等级"
        placeholder: "%player_level%"
        default-operator: ">="
      job:
        name: "职业"
        placeholder: "%player_job%"
        default-operator: "="

队长设置：

text
/gdt team req add power >= 50000
/gdt team req add level >= 60
/gdt team req add job 战士

玩家申请时，插件读取申请者自己的 PAPI 值来判断。

数值条件 ​

战力、等级、转生次数、装备评分适合数值比较。

text
/gdt team req add power >= 50000
/gdt team req add level >= 80
/gdt team req add rebirth >= 3

常用比较符：

text
>=  大于等于
>   大于
<=  小于等于
<   小于
=   等于
!=  不等于

文本条件 ​

职业、阵营、称号、会员组适合文本比较。

text
/gdt team req add job 战士
/gdt team req add guild 青龙会
/gdt team req add title contains 神谕

contains 表示包含。
!contains 表示不包含。

清理条件 ​

查看当前条件：

text
/gdt team req list

移除某一项：

text
/gdt team req remove power

清空全部：

text
/gdt team req clear

队伍开关 ​

text
/gdt team applymode on
/gdt team pvp off
/gdt team loot on
开关配置默认值说明
申请team.default-allow-apply是否接收入队申请
PVPteam.default-allow-pvp是否允许队友伤害
掉落分配team.default-random-distribution是否启用队伍掉落分配

掉落分配还受全局配置影响：

yaml
loot:
  enabled: true
  require-team-toggle: true

如果 require-team-toggle: true，队伍自己的开关也必须开启。

成员跨服状态 ​

跨服网络中，玩家上线时会写入最近所在服务器。队伍界面和公开房间可以显示成员所在服。

成员变量：

text
<member_1_name>
<member_1_server>
<member_1_online>

PAPI 变量：

text
%gtd_team_member_1_name%
%gtd_team_member_1_server%
%gtd_team_member_1_online%

这些变量适合写在 TAB、计分板、聊天格式、客户端 UI 中。
