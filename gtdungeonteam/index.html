GTDungeonTeam 快速开始 ​

GTDungeonTeam 是副本组队和普通组队插件。服务器有 DungeonPlus、QueDP、BTDungeons、MythicDungeons 时，可以把队伍流程接到副本启动、结算、体力和消耗上；服务器暂时没有副本插件时，也能只用队伍、公开房间、邀请、申请、掉落分配、经验共享和竞拍。

作者：RusBrook

先理解它管什么 ​

模块用途
队伍创建队伍、邀请玩家、公开房间、申请入队、队长管理
副本统一读取副本文件，再交给对应副本插件或命令适配器启动
体力每名玩家有体力，上限、初始值、恢复量都能写变量和公式
入队条件配置条件模板，队长在游戏里设置具体要求
进入条件副本文件里写等级、职业、战力、权限等开本条件
成本开本时扣体力、金币、物品，可按队长或全队在线成员扣除
掉落分配队伍内随机、轮询、队长、拾取者分配掉落物
掉落竞拍稀有物品进入竞拍，结束后扣款、给物品、队伍分红、系统抽成
经验共享玩家获得经验时，按队伍人数给队友额外共享经验
客户端界面原版 GUI、PaiUI、AX、DragonCore、KongCore、Dialog 路由
跨服Velocity、BungeeCord、Waterfall 网络中传送队伍并在目标服开本

安装步骤 ​

  1. 安装 GTCore。
  2. 把 GTDungeonTeam.jar 放进 Bukkit、Spigot、Paper 或 Purpur 服务端的 plugins 文件夹。
  3. 如果要用 PlaceholderAPI 变量，安装 PlaceholderAPI。
  4. 如果要接副本插件，安装对应副本插件。
  5. 如果要跨服，代理端也放对应的 GTDungeonTeam 代理插件。
  6. 启动服务器一次，让插件生成配置。
  7. 修改 plugins/GTDungeonTeam/config.yml。
  8. 在 plugins/GTDungeonTeam/dungeons/ 里给每个副本单独写一个 yml。
  9. 输入 /gdt reload 重载。
  10. 玩家输入 /gdt 打开主界面。

/gtdungeonteam、/gdteam、/gdt 是同一个命令。

最小可用配置 ​

刚装好时可以先不接副本插件，只测试队伍。

yaml
team:
  default-max-players: 4
  default-public: true
  default-allow-apply: true

loot:
  enabled: true
  mode: random
  require-team-toggle: true
  require-active-run: false

storage:
  mode: local

ui:
  default: vanilla

重载：

text
/gdt reload

测试：

text
/gdt
/gdt team create
/gdt team invite 玩家名
/gdt rooms list

如果服务器没有副本插件，这一步仍然能正常使用。玩家可以组队、公开房间、申请、邀请，也能开队伍掉落分配。

第一个副本文件 ​

创建：

text
plugins/GTDungeonTeam/dungeons/forest.yml

写入：

yaml
dungeon:
  id: forest
  name: "&a森林遗迹"
  provider: command
  engine-id: forest
  target-server: ""
  min-players: 1
  max-players: 4
  icon: ENDER_EYE
  description:
    - "&7第一章测试副本。"

  requirements:
    scope: leader
    list:
      - "{calc:%player_level%+0} >= 10"

  cost:
    scope: leader
    stamina: 10
    money: 100
    items: []

  start-actions:
    - "[console] say <leader> 开始了 <dungeon_name>"
  reward-actions:
    - "[message] &a你完成了 <dungeon_name>。"

重载后测试：

text
/gdt reload
/gdt team create
/gdt dungeon open forest
/gdt dungeon start

provider: command 适合先打通流程。接 BTDungeons、MythicDungeons 等副本插件时，再把 provider 和 engine-id 改成真实副本。

推荐测试顺序 ​

  1. 先用 storage.mode: local 单服测试队伍。
  2. 确认 /gdt、创建队伍、邀请、公开房间能用。
  3. 写一个 provider: command 的测试副本。
  4. 确认体力、金币、物品消耗按预期扣除。
  5. 再改成真实副本适配器。
  6. 单服流程通了，再接 MySQL、Redis 和跨服代理。
  7. 最后再换 PaiUI、AX、DragonCore、KongCore 或 Dialog。

不要一上来就把跨服、客户端 UI、副本插件、竞拍、多货币全部打开。组队流程是底座，底座先稳，后面好排错。

常用命令 ​

text
/gdt
/gdt help
/gdt help team
/gdt help dungeon
/gdt help rooms
/gdt reload
/gdt status
/gdt providers

/gdt status 只有管理权限能用，会显示存储、代理、队伍、副本适配器、界面、HUD、掉落分配和经验共享状态。

常见第一步问题 ​

现象处理
/gdt 没有命令看插件是否正常启用，先查控制台启动报错
能打开界面但没有副本检查 plugins/GTDungeonTeam/dungeons/ 里是否有 yml，执行 /gdt providers
玩家点开始提示没有队伍先 /gdt team create，队员要先接受邀请或申请加入
队长点开始提示未选择副本先 /gdt dungeon open 副本ID 或在副本界面选择
体力显示不对看 stamina.max、stamina.initial、PAPI 是否能解析
公开房间看不到队伍队伍必须公开，且 team.public-room.enabled 为 true
跨服不动先单服测通，再检查 proxy.enabled、proxy.type、server-id、代理端插件
