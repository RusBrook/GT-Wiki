GTPay 快速开始 ​

GTPay 用于在 Minecraft 服务器内创建微信、支付宝收款订单。玩家下单后，收款码会显示在手中地图上；到账后自动执行点券或礼包奖励命令。

支持 Minecraft 1.12.2 到当前最新版。

GTPay支持本地、MySQL 和 Redis 三种订单存储。多台独立服务端使用同一收款主体和共享存储时，累计赞助、排行、礼包限制和领取记录可以统一统计。

使用前准备 ​

服主需要准备：

  • 已正常运行并完成授权的 GTCore。
  • 有效的 GTPay 使用权限。
  • 一台用于运行 GT收款守护 的安卓设备。
  • 微信、支付宝个人收款码图片。
  • 用于发放点券、权限组、物品或其他奖励的控制台命令。

GTPay 不需要单独放置插件 JAR。服务器启动后由 GTCore 自动加载，配置目录仍然位于：

text
plugins/GTPay/

开通商户 ​

使用购买 GTPay 时登记的 QQ 私聊机器人：

text
创建GTPay 生存一区

机器人会返回本服务器使用的：

text
merchant_id
server_id
server_key
monitor_key

一个 QQ 可以创建多个服务器。不同服务器使用不同名称，例如：

text
创建GTPay 生存一区
创建GTPay 生存二区
创建GTPay RPG一区

忘记密钥时私聊机器人：

text
查询GTPay
查询GTPay 生存一区

同名服务器重复创建会返回原来的配置，不会自动更换密钥。

第一次配置 ​

  1. 启动 Minecraft 服务器。
  2. 在服务器控制台输入：
text
gtpay web open
  1. 在运行服务端的机器上打开控制台返回的本地管理页。
  2. 填写 merchant_id、server_id、server_key、monitor_key。
  3. 保存配置。
  4. 把收款码放入：
text
plugins/GTPay/qrcode/wechat.png
plugins/GTPay/qrcode/alipay.png
  1. 在安卓设备启动 GT收款守护。
  2. App 中填写 merchant_id 和 monitor_key。
  3. 授予通知使用权，确认 App 显示连接成功。

管理页只用于本机配置，默认不会对外开放。配置完成后会在一段时间无操作后自动关闭，也可以执行：

text
gtpay web close

第一个点券订单 ​

先在 topup.yml 中把奖励命令改成服务器实际使用的点券命令，例如：

yaml
name: "点券充值"
enabled: true
scaled: 1.0
integer_only: false
min_amount: 0.01
max_amount: 1000000
permission: ""
commands:
  - "[console] points give {player} {scaled_amount}"

保存后执行：

text
/gtpay reload

玩家创建 10 元微信订单：

text
/gtpay buy 10 wechat

创建 10 元支付宝订单：

text
/gtpay buy 10 alipay

订单有效时间固定为 5 分钟。玩家手持付款地图时按 Q 会取消订单，执行 /gtpay close 也会取消当前订单。

支付金额规则 ​

没有相同待支付订单时，10 元订单就是 10.00 元。

同一服务器、同一渠道、同一基础金额已经存在待支付订单时，GTPay 从最低可用分币开始分配：

text
10.00
10.01
10.02
10.03

已支付、已完成、已取消、已过期和异常订单不会继续占用金额。不是随机增加，也不会无故出现 10.90、1.65 这类大幅加价。

配置文件 ​

首次启动后主要使用以下文件：

text
plugins/GTPay/
  config.yml
  gateway.yml
  message.yml
  topup.yml
  gifts.yml
  sponsorship.yml
  qrcode/
    wechat.png
    alipay.png
  gui/
    main.yml
    gifts.yml

各文件用途：

文件用途
config.yml本地管理页、查询频率、支付渠道、收款码文件和存储方式
gateway.yml商户与服务器密钥，可通过本地管理页填写
message.yml玩家提示与帮助文本
topup.yml点券充值及其发放命令
gifts.yml独立礼包与固定价格
sponsorship.yml永久、周、月、活动累计赞助奖励
gui/*.yml多个自定义充值和礼包 GUI
