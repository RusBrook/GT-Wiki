Lite 与付费版区别 ​

GTMobDropsLite 用来提供免费轻量体验；GTMobDrops 付费版面向大量怪物、复杂掉落池、多套装暗改、活动加成和大型配置管理。

功能对比 ​

功能GTMobDropsLiteGTMobDrops
MythicMobs 内写 GTDrops支持支持
原版物品掉落支持支持
物品库掉落支持常用物品库支持更完整生态
drops 独立文件夹不支持支持
droppools 掉落池不支持支持
多级子文件夹分类不支持支持
套装暗改仅一套多套
公式暗改支持一套内公式完整支持
属性加成一个掉落概率属性完整加成系统
rateBoost不支持支持
独立物品加成不支持支持
本地掉落记录支持支持
MySQL / 数据库存储不支持支持
大量怪物配置维护一般更适合

Lite 适合什么 ​

  • 只想在 MythicMobs 怪物里直接写掉落。
  • 怪物数量不多。
  • 不需要掉落池复用。
  • 不需要多套套装。
  • 不需要按活动、权限、区域、物品做复杂倍率。
  • 想先体验 GT 系列掉落写法。

付费版适合什么 ​

  • 怪物很多，掉落配置需要分文件夹管理。
  • 多个怪物共用同一组掉落。
  • 想做 Boss、普通怪、活动怪不同掉落池。
  • 想做多套装备暗改。
  • 想做属性、权限、PAPI、活动、区域等高级加成。
  • 想让掉落配置长期维护不混乱。

配置迁移 ​

Lite：

yaml
ForestElite:
  Type: ZOMBIE
  GTDrops:
    drops:
      - "OAK_LOG 8-16 100"
      - "ni:forest_sword 1 3"

升级付费版后可以继续保留，也可以迁移到独立文件夹：

text
plugins/GTMobDrops/drops/
└─ forest/
   └─ elite.yml

付费版可以把大量掉落拆开维护：

yaml
ForestElite:
  drops:
    - forest/elite
  pools:
    - forest/common

常见选择 ​

小型测试服 ​

使用 Lite：

yaml
GTDrops:
  drops:
    - "DIAMOND 1 10"

简单、直接、文件少。

RPG 正式服 ​

使用 GTMobDrops：

text
drops/
├─ boss/
├─ normal/
└─ event/

droppools/
├─ forest/
├─ dungeon/
└─ worldboss/

掉落可以分类、复用、组合，后期维护成本低。

套装追赶机制 ​

Lite 可以做一套：

yaml
Sets:
  legend_set:
    items:
      - "ni:legend_helmet"
      - "ni:legend_chestplate"
    darkDrop:
      enabled: true
      ownedModifier: "-2%"
      missingModifier: "inverse"

需要多套职业装备、多个副本套装、多个阶段套装时，使用付费版。

升级建议 ​

从 Lite 开始时，建议遵守两点：

  1. 怪物内统一使用 GTDrops。
  2. 物品 ID 统一使用 GTCore 物品 ID，例如 DIAMOND、ni:item_id、mm:item_id。

这样升级到付费版时，不需要推翻原配置，只需要把复杂内容逐步拆到独立文件夹和掉落池。
