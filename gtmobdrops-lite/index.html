GTMobDropsLite 快速开始 ​

GTMobDropsLite 是免费轻量版掉落插件，核心玩法是直接在 MythicMobs 怪物配置里写 GTDrops。
它保留物品库支持、基础属性加成、本地掉落记录和一套套装暗改，适合轻量服务器或付费版前的体验环境。

安装 ​

  1. 安装 GTCore。
  2. 安装 MythicMobs。
  3. 放入 GTMobDropsLite。
  4. 在 GTCore 配置中填写 GTMobDropsLite 激活码。
  5. 启动服务器生成 plugins/GTMobDropsLite/。
  6. 在 MythicMobs 怪物配置中添加 GTDrops。

重载：

text
/gtmobdropslite reload

文件位置 ​

text
plugins/GTMobDropsLite/
├─ config.yml
├─ sets.yml
└─ logs/
   └─ drops.log
文件用途
config.yml属性加成、本地记录、基础运行设置
sets.ymlLite 保留的一套套装暗改
logs/drops.log本地掉落记录

GTDrops 写在哪里 ​

写在 MythicMobs 的怪物节点里：

yaml
LiteTestMob:
  Type: ZOMBIE
  Display: '&aLite 测试怪'
  Health: 50
  Damage: 3
  GTDrops:
    drops:
      - "DIAMOND 1 100"

效果：击杀 LiteTestMob 后必定掉落 1 个钻石。

概率写法 ​

Lite 的 GTDrops 概率按百分数写：

写法含义
100100%
5050%
1010%
0.50.5%
10%10%

推荐直接写整数百分数，最清楚：

yaml
GTDrops:
  drops:
    - "DIAMOND 1 10"

效果：10% 概率掉落 1 个钻石。

数量写法 ​

固定数量：

yaml
GTDrops:
  drops:
    - "DIAMOND 3 100"

效果：必定掉落 3 个钻石。

随机区间：

yaml
GTDrops:
  drops:
    - "GOLD_INGOT 1-3 50"
    - "IRON_INGOT 2..5 30"
    - "EMERALD 1~2 10"

支持 1-3、1..3、1~3 三种区间写法。

原版物品 ​

原版物品直接写材料 ID：

yaml
GTDrops:
  drops:
    - "DIAMOND 1 10"
    - "GOLD_INGOT 2-4 25"
    - "NETHER_STAR 1 1"

效果：

  • 10% 掉 1 个钻石。
  • 25% 掉 2 到 4 个金锭。
  • 1% 掉 1 个下界之星。

物品库物品 ​

使用 GTCore 统一物品 ID：

yaml
GTDrops:
  drops:
    - "ni:legend_helmet 1 5"
    - "mm:BossKey 1 20"
    - "ia:namespace:item_id 1 10"
    - "oraxen:my_item 1 10"
    - "sx:my_item 1 10"

常用前缀：

前缀插件
ni:NeigeItems
mm:MythicMobs 物品
ia:ItemsAdder
oraxen: / oa:Oraxen
sx:SX-Item

如果物品库没安装或物品不存在，该条掉落会跳过，不会影响其他掉落。

detailedDrops 写法 ​

想让配置更清楚时，用 detailedDrops：

yaml
LiteTestMob:
  Type: ZOMBIE
  Display: '&aLite 测试怪'
  Health: 50
  Damage: 3
  GTDrops:
    detailedDrops:
      diamond:
        id: "DIAMOND"
        amount: "1"
        chance: 100
      boss_key:
        id: "mm:BossKey"
        amount: "1"
        chance: 20
      legend_helmet:
        id: "ni:legend_helmet"
        amount: "1"
        chance: 5

也可以用 item：

yaml
GTDrops:
  detailedDrops:
    reward:
      item: "DIAMOND"
      amount: "1-3"
      chance: 50

也支持 rate、probability 作为概率字段：

yaml
GTDrops:
  detailedDrops:
    reward:
      item: "DIAMOND"
      amount: "1"
      rate: 10

同时写多个掉落 ​

yaml
ForestElite:
  Type: ZOMBIE
  Display: '&2森林精英'
  Health: 300
  Damage: 8
  GTDrops:
    drops:
      - "OAK_LOG 8-16 100"
      - "EMERALD 1-2 25"
      - "ni:forest_sword 1 3"
      - "mm:ForestKey 1 10"

效果：

  • 木头必掉 8 到 16 个。
  • 绿宝石 25% 概率掉 1 到 2 个。
  • NI 森林剑 3% 概率掉 1 个。
  • MM 森林钥匙 10% 概率掉 1 个。

属性掉落概率加成 ​

Lite 保留一个属性加成：读取玩家身上的指定属性，把它作为额外掉落概率加成。

yaml
AttributeBonus:
  enabled: true
  dropChanceAttribute: "掉落概率"
  maxDropChance: 100

含义：

字段说明
enabled是否启用属性加成
dropChanceAttribute属性插件里的属性名
maxDropChance最大额外加成，100 表示最多 100%

例子：

  • 玩家 掉落概率 = 5，表示额外加成 5%。
  • 玩家 掉落概率 = 30，表示额外加成 30%。
  • maxDropChance: 100 时，最高不会超过 100% 额外加成。

这个属性由 GTCore 统一读取，可接入 AttributePlus、CraneAttribute、SX-Attribute 等属性来源。

本地掉落记录 ​

yaml
Storage:
  enabled: true
  file: "logs/drops.log"
  flush-interval-ticks: 100

记录内容包括玩家、怪物、物品 ID、数量和时间。
Lite 不使用 MySQL，只保留本地文件记录。

一套套装暗改 ​

Lite 只读取 sets.yml 中的第一套套装。
示例：

yaml
Sets:
  legend_set:
    name: "传说套装"
    items:
      - "ni:legend_helmet"
      - "ni:legend_chestplate"
      - "ni:legend_leggings"
      - "ni:legend_boots"
      - "ni:legend_sword"
    darkDrop:
      enabled: true
      ownedModifier: "-2%"
      missingModifier: "inverse"
      formula:
        enabled: false
        owned: "{drop_rate} * (1 - 0.02 * {owned_count})"
        missing: "{drop_rate} * (1 + 0.01 * {missing_count})"
    bonusRate: "5%"
    showTip: true
    tipMessage: "&e[套装] &7{set_name}: &a+{bonus}%"
    completeMessage: "&6&l恭喜！&r &e你已集齐 &6{set_name} &e全套！"

逻辑：

  • 玩家已获得的部件降低掉率。
  • 玩家未获得的部件提高掉率。
  • missingModifier: "inverse" 表示未获得部件使用 ownedModifier 的相反值。
  • bonusRate 是整套物品的额外基础加成。
  • formula.enabled: true 时，使用公式计算，优先级高于固定值。

公式模式 ​

yaml
darkDrop:
  enabled: true
  formula:
    enabled: true
    owned: "{drop_rate} * (1 - 0.02 * {owned_count})"
    missing: "{drop_rate} * (1 + 0.01 * {missing_count})"

可用变量：

变量内容
{drop_rate}基础掉率
{owned_count}已获得套装部件数量
{missing_count}未获得套装部件数量
{item_id}当前判断的物品 ID

例子：玩家已经获得 2 件套装，当前物品基础掉率为 10%。
owned 公式会变成：

text
0.10 * (1 - 0.02 * 2) = 0.096

结果：已拥有部件掉率降到 9.6%。

从 Lite 升级到付费版 ​

建议 Lite 也从一开始使用 GTDrops 节点。以后升级到 GTMobDrops 付费版时：

  1. MythicMobs 怪物里的 GTDrops 可以继续保留。
  2. 简单掉落继续放在怪物内。
  3. 大量重复掉落迁移到 drops/ 文件夹。
  4. 多怪共用掉落迁移到 droppools/。
  5. 多套套装和高级加成迁移到付费版完整系统。

排查 ​

不掉落 ​

检查：

  1. MythicMobs 怪物 ID 是否正确。
  2. GTDrops 是否写在怪物节点下。
  3. 是否写成 drops 或 detailedDrops。
  4. 概率是否过低。
  5. 物品 ID 是否能被 GTCore 识别。

先用 100% 原版物品测试：

yaml
GTDrops:
  drops:
    - "DIAMOND 1 100"

物品库物品不掉 ​

检查：

  1. 对应物品库插件是否启用。
  2. GTCore 是否识别到该适配器。
  3. 物品 ID 是否和物品库配置一致。
  4. 先用 /give 或物品库自身命令确认物品能生成。

套装暗改没效果 ​

检查：

  1. sets.yml 的 Sets 下是否有套装。
  2. Lite 只读取第一套。
  3. items 里的物品 ID 是否和掉落 ID 一致。
  4. 本地掉落记录是否开启。
  5. 玩家是否已经通过掉落获得过该部件。
