多套 GUI ​

GTGem 可以分别创建宝石、符文、附魔石等界面。每套 GUI 都能单独设置标题、布局和按钮内容，并通过对应的配置 ID 打开。

文件位置 ​

原有配置继续保留：

text
plugins/GTGem/gui.yml

它的配置 ID 固定为 default。额外 GUI 放在：

text
plugins/GTGem/guis/

目录支持任意层级：

text
plugins/GTGem/guis/rune.yml
plugins/GTGem/guis/enchant.yml
plugins/GTGem/guis/server-a/rune-board.yml

对应 ID：

text
rune
enchant
server-a/rune-board

修改后执行：

text
/gtgem reload

打开指定 GUI ​

text
/gtgem open rune
/gtgem remove rune
/gtgem info rune

/gtgem open enchant
/gtgem remove enchant
/gtgem info enchant

管理员为其他在线玩家打开：

text
/gtgem open rune 玩家名

继承 default ​

额外 GUI 不需要复制整份 gui.yml。没有填写的视图、布局和按钮会继承 default。

最小符文界面：

yaml
embed:
  title: "&8[ &d符文镶嵌 &8]"
  items:
    gem:
      material: FIRE_CHARGE
      name: "&d[ &5放入符文 &d]"
      lore:
        - "&7点击背包中的符文放入"

remove:
  title: "&8[ &c符文取出 &8]"

info:
  title: "&8[ &d符文查看 &8]"

这份配置只改了标题和符文按钮，尺寸、槽位矩阵、确认按钮和边框都继承 gui.yml。

完整布局 ​

需要独立排版时，可以覆盖 size、layout 和 matrix：

yaml
embed:
  title: "&8[ &6附魔台 &8]"
  size: 54
  layout:
    equipment: "E"
    gem: "G"
    socket: "S"
    confirm: "C"
    info: "I"
    border: "B"
    empty: " "
  matrix:
    - "BBBBBBBBB"
    - "B       B"
    - "B E G I B"
    - "B  SSS  B"
    - "B  SSS  B"
    - "B   C   B"
  items:
    equipment:
      material: DIAMOND_CHESTPLATE
      name: "&e放入装备"
      lore:
        - "&7点击背包中的装备放入"
    gem:
      material: ENCHANTED_BOOK
      name: "&6放入附魔石"
      lore:
        - "&7点击背包中的附魔石放入"
    confirm:
      material: ANVIL
      name: "&a确认附魔"
      lore:
        - "&7成功率: &e{chance}%"
        - "&7金币: &f{money}"
        - "&7经验: &f{exp}"

matrix 每行最多 9 个字符，最多 6 行。空格不会自动填充物品。

变量 ​

名称和 Lore 支持 PAPI，也支持以下 GTGem 变量：

变量内容
{player}玩家名
{equipment_name}当前装备名
{gem_name}当前宝石名
{socket_type}当前槽位类型
{chance}成功率百分比
{money}镶嵌金币消耗
{exp}镶嵌经验消耗
{remove_money}拆卸金币消耗
{remove_exp}拆卸经验消耗
{remove_items}拆卸物品消耗
{attribute_list}宝石属性，多行展开
{condition_list}镶嵌条件，多行展开
{remove_condition_list}拆卸条件，多行展开
