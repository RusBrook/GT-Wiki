主配置 config.yml ​

plugins/GTGather/config.yml 控制采集系统的全局行为：调试、冷却、触发范围、全息显示、采集工具、读条、限制、奖励模式和提示消息。

采集点的具体内容通常写在：

text
plugins/GTGather/templates.yml
plugins/GTGather/templates/*.yml
plugins/GTGather/spawns.yml

修改 config.yml 后执行：

text
/gtgather reload

涉及实体显示模式、发包显示或依赖插件时，建议重启服务器。

debug 调试 ​

yaml
debug:
  enabled: false
  verbose: false
  log-spawn: false
  log-cast: false
  log-wand: false
  log-gui: false
  log-trigger: false
  log-reward: false
  log-cooldown: false
  log-hook: false
  log-config: false
  log-performance: false
  log-entity: false
  log-protection: false
字段作用
enabled调试总开关
verbose输出更详细信息
log-spawn生成点加载和创建日志
log-cast采集读条日志
log-wand工具操作日志
log-guiGUI 操作日志
log-trigger条件检测日志
log-reward奖励发放日志
log-cooldown冷却判定日志
log-hook物品库、PAPI 等 Hook 日志
log-performance性能日志
log-entity实体显示日志
log-protection奖励保护日志

排查采集点不能采集：

yaml
debug:
  enabled: true
  log-trigger: true
  log-cast: true
  log-reward: true

排查完成后关闭。

collect-cooldown 全局冷却 ​

yaml
collect-cooldown:
  enabled: true
  default: 1.0
  bypass-permission: "gtgather.bypass.cooldown"
字段作用
enabled是否启用采集冷却
default默认冷却秒数
bypass-permission绕过冷却权限

示例：默认每个采集点采完后 3 秒内不能再次采集：

yaml
collect-cooldown:
  enabled: true
  default: 3.0

模板里的 cast.cooldown 可以覆盖全局冷却。

trigger 触发范围 ​

yaml
trigger:
  range: 4.5
  require-sneak: false
字段作用
range玩家距离采集点多远可以触发
require-sneak是否要求潜行

副本机关类采集点可以要求潜行：

yaml
trigger:
  range: 3.0
  require-sneak: true

hologram 显示 ​

yaml
hologram:
  mode: auto
  height-offset: 2.5
  refresh-interval: 20
  show-template-id: true
字段作用
mode显示模式，auto 自动选择
height-offset显示高度偏移
refresh-interval刷新间隔，20 tick = 1 秒
show-template-id是否显示模板 ID

采集点文字太高：

yaml
hologram:
  height-offset: 2.0

采集点文字刷新不够及时：

yaml
hologram:
  refresh-interval: 10

刷新越频繁越吃性能，不建议低于 5。

wand 采集点创建工具 ​

yaml
wand:
  enabled: true
  item:
    material: BLAZE_ROD
    name: "&6&l采集点创建工具"
    lore:
      - "&7右键地面快速创建生成点"
      - "&7Shift+右键打开配置面板"
      - "&7左键空气快速选择模板"
      - "&e§o通过 /gtgather wand 获取"
  id-mode: AUTO
  id-format: "spawn_{player}_{counter}_{random4}"
字段作用
enabled是否启用创建工具
item.material工具材质
item.name工具名称
item.lore工具说明
id-mode生成点 ID 模式
id-format自动创建生成点时使用的 ID 模板

id-format 支持：

text
{player}
{player_uuid}
{template}
{template_id}
{counter}
{date}
{time}
{random4}
{random8}
{uuid4}

安装 PlaceholderAPI 后也可以写：

text
%player_name%
%player_world%

ID 支持中文；空格会自动转成下划线，斜杠和控制字符会被过滤。

示例：让生成点 ID 带模板 ID：

yaml
wand:
  id-format: "{template_id}_{player}_{counter}_{random4}"

玩家选择模板 T3_采集点_木材_寒松木残枝 后创建，ID 会类似：

text
T3_采集点_木材_寒松木残枝_RusBrook_1_A7F2

wand.behavior 工具动作 ​

yaml
wand:
  behavior:
    on-shift-right-click: "open-config-gui"
    on-right-click-ground: "create-spawn"
    on-left-click-air: "quick-select-template"
动作说明
open-config-gui打开配置面板
create-spawn在点击位置创建生成点
quick-select-template快速选择模板

默认行为已经适合大多数使用场景。

cast 采集读条 ​

yaml
cast:
  enabled: true
  default-duration: 3.0
  lock-movement: true
  lock-interact: true
  allow-shift-cancel: true
  progress-bar: BOSS_BAR
  progress:
    filled: "▌"
    empty: "▌"
    filled-color: "&a"
    empty-color: "&7"
字段作用
enabled是否启用采集读条
default-duration默认采集时长，秒
lock-movement采集中移动是否取消
lock-interact采集中交互是否取消
allow-shift-cancel是否允许 Shift 取消
progress-bar进度条显示方式

进度条类型：

text
ACTION_BAR
BOSS_BAR
TITLE

例如想做 5 秒采集，移动取消：

yaml
cast:
  enabled: true
  default-duration: 5.0
  lock-movement: true

模板里的 cast.duration 可以覆盖全局默认时长。

limit 采集上限 ​

yaml
limit:
  enabled: true
  default-reset: "1d"
  default-amount: 50
  bypass-permission: "gtgather.bypass.limit"
字段作用
enabled是否启用采集上限
default-reset默认重置周期
default-amount默认采集次数
bypass-permission绕过上限权限

时间写法：

text
30s
5m
1h
1d

每名玩家每天默认最多采 100 次：

yaml
limit:
  enabled: true
  default-reset: "1d"
  default-amount: 100

模板里的 limit.player 和 limit.server 可以覆盖全局限制。

reward 奖励发放 ​

yaml
reward:
  default-mode: INVENTORY
  protection:
    enabled: true
    timeout: 60
    namespace: "gtgather"
    owner-key: "owner"
    denied-message: "&c这个物品属于 &e{owner}&c，你不能拾取"

default-mode：

text
INVENTORY  直接进入背包
DROP       掉落到地上

奖励保护用于 DROP 模式，防止其他玩家抢走采集物。

例如掉落奖励 30 秒后可被其他玩家拾取：

yaml
reward:
  default-mode: DROP
  protection:
    enabled: true
    timeout: 30

messages 提示消息 ​

yaml
messages:
  prefix: "&b&lGTG &3» &r"
  gather-success:
    - "&a✦ &e采集获得 &a✦"
    - "&7- &f{display-name} &7× &e{amount}"
  cast-start:
    - "&a开始采集 &e{template-name} &a..."
    - "&7按住 Shift 取消"
  cast-tick: "&a采集中... &7(&f{progress}%&7) &8Shift 取消"

常用变量：

text
{display-name}
{amount}
{template-name}
{progress}
{remaining}
{fail_reason}
{owner}

可以写单行：

yaml
cast-cancel-shift: "&c你取消了采集"

也可以写多行：

yaml
gather-success:
  - "&a采集成功"
  - "&7获得：&f{display-name} &ex{amount}"

最小配置方案 ​

稳定基础配置：

yaml
collect-cooldown:
  enabled: true
  default: 1.0

trigger:
  range: 4.5
  require-sneak: false

cast:
  enabled: true
  default-duration: 3.0
  lock-movement: true
  allow-shift-cancel: true

reward:
  default-mode: INVENTORY

适合大多数普通采集点。

验证流程 ​

  1. 执行 /gtgather reload。
  2. 执行 /gtgather wand 获取工具。
  3. 左键空气选择模板。
  4. 右键地面创建生成点。
  5. 右键采集点，确认读条、冷却、奖励都正常。
  6. 如果失败，开启 debug.log-trigger 和 debug.log-reward 排查。
