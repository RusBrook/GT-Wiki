配置文件怎么读 ​

GT 系列插件大部分配置都是 yml 文件。只要记住缩进、冒号、列表和概率这几件事，就能看懂大部分教程。

文件放在哪里 ​

插件配置一般在：

text
plugins/插件名/

例如：

text
plugins/GTCore/config.yml
plugins/GTMobDrops/config.yml
plugins/GTMobDrops/drops/xxx.yml
plugins/GTGem/config.yml
plugins/GTGather/config.yml

教程里写 plugins/GTMobDrops/drops/novice/zombie.yml，意思就是在服务器目录下找到 plugins 文件夹，然后按这个路径创建文件夹和文件。

缩进 ​

YAML 用缩进表示上下级。

正确：

yaml
ZOMBIE:
  enabled: true
  drops:
    - "DIAMOND 1 1.0"

不推荐：

yaml
ZOMBIE:
enabled: true
drops:
- "DIAMOND 1 1.0"

建议统一用两个空格缩进，避免使用 Tab。

冒号后面要有空格 ​

正确：

yaml
enabled: true
chance: 0.1
name: "&a测试物品"

不推荐：

yaml
enabled:true
chance:0.1
name:"&a测试物品"

列表 ​

列表每一行前面有 -。

yaml
drops:
  - "DIAMOND 1 1.0"
  - "GOLD_INGOT 3 0.5"

这表示两条掉落：

  • 第一条：钻石。
  • 第二条：金锭。

如果列表里每一项有很多字段，可以写成对象：

yaml
detailed-drops:
  rare_gem:
    id: "ni:rare_gem"
    amount: 1
    chance: 0.05

概率必须看插件说明 ​

概率字段按对应插件页面的说明填写。GT 系列里不同字段有不同含义，教程会在对应页面说明。

GTMobDrops 掉落概率使用 0.0 ~ 1.0：

text
1.0  = 100%
0.5  = 50%
0.1  = 10%
0.05 = 5%
0.01 = 1%

所以 GTMobDrops 里 100% 掉钻石应该写：

yaml
drops:
  - "DIAMOND 1 1.0"

10% 掉钻石应该写：

yaml
drops:
  - "DIAMOND 1 0.1"

GTGem 的镶嵌成功率也按 0.0 ~ 1.0 理解：

yaml
rules:
  chance: 0.8

这里表示 80% 成功率。

属性值和掉落字段概率 ​

有些属性插件里会写：

text
掉落概率: 5

这个值由插件配置里的属性加成规则换算，通常表示玩家身上有 5 点掉落概率属性。

写怪物物品掉落时，仍然按 GTMobDrops 的概率口径：

yaml
drops:
  - "DIAMOND 1 0.05"

这里才是 5% 掉落。

文本建议加引号 ​

带颜色代码、中文、冒号、百分号、PAPI 变量的文本，建议加引号。

yaml
name: "&6传说套装"
bonusRate: "5%"
message: "&a你获得了 &e{item}"
condition: "papi:%player_level% >= 30"

改完怎么让配置生效 ​

大部分插件都有重载命令。

text
/gtcore reload
/gtmd reload
/gtgem reload
/gtgather reload
/gtcdk reload

如果你改了前置、依赖、脚本开关、数据库连接、授权配置，建议直接重启服务器。
如果只是改掉落、宝石、采集模板、GUI 文本，通常重载就够。
