依赖与版本 ​

GT 系列会尽量兼容多版本服务端和多种生态插件，但每个服务器环境都不同。安装前先理解依赖关系，后面排错会轻松很多。

服务端核心 ​

推荐：

  • Paper
  • Purpur
  • Spigot 兼容核心

不建议在正式服直接使用非常冷门或魔改严重的核心。
如果一定要用，先在测试服跑完整功能。

Java 版本 ​

低版本 Minecraft 往往使用 Java 8，高版本通常使用 Java 17 或更高。
如果你在 1.12.2 这类老版本使用插件，要特别注意：

  • 插件是否使用了过高版本 Java 编译。
  • 依赖库是否支持 Java 8。
  • 脚本引擎是否会加载不兼容的实现。

软依赖 ​

软依赖不是每个服都必须安装，但装了以后插件会自动接入更多功能。

类型常见插件GT 系列用途
怪物MythicMobs怪物掉落、MM 物品、Boss 玩法
物品库NeigeItems、ItemsAdder、Oraxen、CraftEngine读取自定义物品
属性AttributePlus、CraneAttribute、SX-Attribute掉落加成、宝石属性、伤害显示
变量PlaceholderAPI条件、显示文本、动态倍率
显示Adyeshach、ProtocolLib、PacketEvents实体显示、血条、发包能力
经济Vault、XConomy金币扣除、兑换、宝石成本

GTQuiver 依赖说明 ​

GTQuiver 硬依赖 GTCore，自带 TabooLib 6.3 运行内容。插件字节码兼容 Java 8，目标服务端版本为 1.12.2 至 1.21.11。

可选接入：

插件用途是否必装
CraneAttribute将箭袋中实际发射箭的 CA 属性用于本次远程伤害否
AttributePlus Pro将箭袋中实际发射箭的 AP 属性用于本次远程伤害否
MythicMobs自定义箭命中时施放技能否
GTCore 已支持的物品库用 provider-item 创建箭袋或自定义箭否

CA 与 AP 同时安装时，由 config.yml 的 attribute-bridge.provider 明确选择。默认是 CA，AP 服务器必须改为 AP；不要把 BOTH 当成自动检测选项。

安装顺序建议 ​

text
基础依赖 / 生态插件
        ↓
GTCore
        ↓
GT 系列业务插件
        ↓
修改配置并重启

如果业务插件在 GTCore 前加载，可能会出现找不到 API、命令注册失败、适配器未初始化等问题。

检查依赖是否启用 ​

进入服务器后输入：

text
/plugins

确认相关插件是绿色。
如果某个依赖是红色，先修它，不要继续排 GT 插件。
