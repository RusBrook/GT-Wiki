安装路线 ​

本页按推荐安装顺序整理。建议先跑通最小闭环，再逐步增加插件和复杂配置。

第一步：准备服务端 ​

推荐使用 Paper 或 Purpur。
版本根据插件目标选择，GT 系列会尽量覆盖常见版本，但低版本服端需要额外注意 Java 和依赖兼容。

建议先确认：

  • 服务端可以正常启动。
  • plugins 文件夹存在。
  • 控制台没有其他插件刷屏报错。
  • 服务器能访问外网，用于下载依赖或验证。

第二步：安装基础依赖 ​

不同插件需要的依赖不完全一样，下面是常见组合：

依赖用途建议
PlaceholderAPI变量、条件、显示文本推荐安装
Vault经济接口需要金币消耗时安装
ProtocolLib发包、显示、兼容使用显示类功能时推荐安装
PacketEvents高版本发包能力需要时安装
MythicMobs怪物、技能、MM 物品做 RPG 基本必装
NeigeItems / ItemsAdder / Oraxen / CraftEngine物品库按服务器实际选择
AttributePlus / CraneAttribute / SX-Attribute属性来源按服务器实际选择

控制首次测试范围

物品库和属性插件可以兼容多个。首次测试建议只保留当前玩法实际需要的插件，先跑通一条链路，再补更多兼容。

第三步：安装 GTCore ​

  1. 将 GTCore.jar 放入 plugins。
  2. 启动服务器一次。
  3. 确认生成 plugins/GTCore/ 文件夹。
  4. 关闭服务器。
  5. 打开 plugins/GTCore/config.yml，填写 QQ 和插件激活码。
  6. 重新启动服务器。

启动成功后，控制台应能看到 GTCore 启动信息。

第四步：安装业务插件 ​

根据你要的玩法安装：

玩法插件
免费测试怪物掉落GTMobDropsLite
完整怪物掉落GTMobDrops
宝石镶嵌GTGem
采集点GTGather
CDK / 兑换GTCDK
伤害数字 / 怪物血条GTDamage

安装方式：

  1. 将插件放入 plugins。
  2. 启动服务器生成配置。
  3. 关闭服务器。
  4. 修改配置。
  5. 重新启动服务器测试。

第五步：跑最小闭环 ​

先让每个插件完成一个最小动作，再继续添加复杂配置。

插件最小测试
GTMobDrops一个 MM 怪物掉落一个原版物品
GTMobDropsLite一个 MM 怪物配置 GTDrops
GTGem一个宝石能镶嵌到一件装备
GTGather一个采集点采集后给一个物品
GTCDK一个 CDK 能兑换一次奖励
GTDamage打怪时能显示伤害数字或血条

检查点 ​

安装完成后，用这个表检查：

  • 控制台没有红色报错。
  • /plugins 能看到插件是绿色启用状态。
  • GTCore 已经先于业务插件启动。
  • 物品库、属性插件、MythicMobs 已经安装。
  • PlaceholderAPI 变量能正常解析。
  • 修改 YAML 后没有缩进错误。

常见错误 ​

插件没有生成配置 ​

通常是插件没有成功启用。排查时从控制台最早的报错开始看。

配置改了没生效 ​

优先完整重启服务器。很多玩法涉及缓存、脚本、物品库和属性读取，热重载不一定能完全刷新。

物品识别不到 ​

先确认对应物品库插件已经启用，再确认物品 ID 写法和文档一致。
